Ceph Command KB

User Guide — Verify Ceph commands, configs, and test scripts from your AI agent

1. What is Ceph Command KB?

Ceph Command KB is an MCP server that provides a verified knowledge base of 1,254 Ceph CLI commands and 2,660 configuration parameters for Ceph 20.2.1 (Tentacle). It eliminates command hallucinations when AI agents generate Ceph automation or tests.

  • Verify commands, flags, and arguments before generating code
  • Search commands by keyword, description, or flag
  • Check config parameters — type, default, min/max, runtime updatability
  • Review test scripts for invalid commands, missing cleanup, and risks

Ships pre-built — no Ceph cluster needed. Supports Cursor, IBM Bob, Claude Desktop, and REST API consumers.

2. Connect Your Agent

Option A: Cursor (stdio — recommended)

Add to ~/.cursor/mcp.json:

{
  "mcpServers": {
    "ceph-cmd-kb": {
      "command": "python3",
      "args": ["-m", "ceph_command_kb.server.mcp_server", "--kb-path", "/path/to/knowledge/ceph-20.2.1-tentacle"],
      "cwd": "/path/to/ceph-command-kb"
    }
  }
}

Restart Cursor. The server starts automatically.

Option B: HTTP/SSE (Bob, Claude Desktop, Continue, Cline)

Start the server:

python -m ceph_command_kb.server.mcp_server --transport sse --host 0.0.0.0 --port 8081

Configure your agent to connect to http://localhost:8081/sse:

{
  "mcpServers": {
    "ceph-cmd-kb": {
      "url": "http://localhost:8081/sse",
      "transport": "sse"
    }
  }
}

For Cursor: ~/.cursor/mcp.json. For Bob: .bob/mcp.json. For Claude Desktop: claude_desktop_config.json.

5. Running All Ceph MCPs Together

Three MCPs form the Ceph Engineering Intelligence Platform:

python -m ceph_doc_kb.server.mcp_server --transport sse --port 8082 &
python -m ceph_issue_kb.server.mcp_server --transport sse --port 8083 &
{
  "mcpServers": {

    "ceph-doc-kb": { "url": "http://localhost:8082/sse", "transport": "sse" },
    "ceph-issue-kb": { "url": "http://localhost:8083/sse", "transport": "sse" }
  }
}

6. Troubleshooting

  • Agent doesn't see the server — Restart the IDE after editing mcp.json. Verify the server name and path match this guide.
  • Commands not verified — Check that --kb-path points to the correct knowledge base directory containing commands.json.
  • Config tools return empty — Verify configs.json exists in the KB directory.
  • SSE connection refused — Verify the server is running: curl http://localhost:8081/health.
